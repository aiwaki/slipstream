# Decisions

Stable decisions and invariants for Slipstream. Add entries when a rule should
survive across sessions and agents.

| Date | Decision | Status |
|---|---|---|
| 2026-07-16 | Shared routing state machines live in the pure `crates/slipstream-core` crate. The core accepts addresses, times, and outcomes as data and owns no sockets, clocks, filesystem, process, Tauri, or OS calls; adapters own those effects. Frozen language-neutral contract tests move with the core, while the tray may re-export modules only for transition compatibility. | Active |
| 2026-07-16 | After automatic context compaction or task resume, agents must verify the current worktree, merged/open PRs, required CI, and `docs/CURRENT_STATE.md` before choosing work. Conversation summaries and milestone labels are navigation only, never completion evidence. A PR that changes the next verified action or milestone status updates the checkpoint in the same change. | Active |
| 2026-07-07 | Discord traffic stays on local bypass/desync, including `updates.discord.com` and `gateway.discord.gg`. It must not route through Geph. | Active |
| 2026-07-07 | YouTube/googlevideo video delivery stays on local bypass. Geph is not the fallback for video playback. | Active |
| 2026-07-07 | QUIC/UDP must not be blocked globally. Any future QUIC handling must be scoped to verified host/IP evidence and a concrete failure mode. | Active |
| 2026-07-08 | Geph is reserved for geo-exit cases where the service rejects the user's Russian IP address. | Active |
| 2026-07-13 | Unknown hosts must never auto-promote to Geph from local stalls plus a successful tunnel payload: that only proves that Geph works, not that a foreign exit is required. Geo-exit is selected only by explicit, reviewed policy. | Active |
| 2026-07-13 | `google.com` and the Spotify delivery families (`spotify.com`, `spotifycdn.com`, `scdn.co`) use a local-only `direct_first` ladder: plain TLS is always tried first, then bounded local desync if necessary; Geph is never an edge. A cached local fallback cannot displace the next direct attempt. | Active |
| 2026-07-08 | Slipstream must not mutate external DNS, VPN, PAC, or proxy settings. It may detect and warn about them. | Active |
| 2026-07-08 | Repo docs are the primary source of project knowledge. Codex memory stores durable agent/user preferences and pointers, not long investigations. | Active |
| 2026-07-09 | GitHub developer/download endpoints use direct passthrough and plain TLS; they should not route through Geph or the generic desync ladder. | Active |
| 2026-07-09 | Human-facing runtime logs should avoid `failed` for expected transient retry/fallback events; reserve alarming wording for action-required errors. | Active |
| 2026-07-10 | Slipstream owns only the `com.apple/slipstream` PF anchor. It must not load Slipstream rules into the global ruleset, edit `/etc/pf.conf`, or disable PF globally. | Active |
| 2026-07-16 | A root PF listing that resembles a pre-anchor Slipstream redirect is not ownership proof. Detection is read-only: the daemon disables only its own launchd label, clears only its private anchor/token, reports the conflict, and exits. Startup, recovery, upgrade, and uninstall never reload `/etc/pf.conf` automatically. | Active |
| 2026-07-16 | A terminal startup `conflict` remains visible after the normal 15-second live-status TTL. Successful startup replaces it and uninstall removes the status file; stale live states still become `off`. | Active |
| 2026-07-10 | PF enablement is reference-counted with the token returned by `pfctl -E`; cleanup flushes only the private anchor and releases only that token. | Active |
| 2026-07-10 | The bundled Geph listener on `127.0.0.1:9954` is usable only when PID, executable, config path, and listener ownership agree. An unknown listener is a fail-closed conflict. | Active |
| 2026-07-10 | A separately managed Geph listener on `127.0.0.1:9909` is diagnostic-only unless the user explicitly opts into that port. Slipstream never stops it. | Active |
| 2026-07-10 | Geph configuration directories are owner-only (`0700`); secret-bearing config, cache, and ownership files are `0600` and written atomically. | Active |
| 2026-07-16 | A legacy plaintext Geph secret is removed only after the exact Slipstream Keychain write succeeds. A failed migration retains the source value for retry; a failed manual account update does not restart the owned Geph job as if it succeeded. A later successful Keychain read retries removal of any stale plaintext copy. | Active |
| 2026-07-10 | A transparent HTTPS PF redirect loaded before `com.apple/*` owns the real traffic path because PF uses the first matching translation. Slipstream must pause, report the conflicting anchor, and recover automatically when it disappears; it must not flush, stop, or rewrite the other product. | Active |
| 2026-07-10 | Runtime recovery is selected by a pure reducer. Discord/YouTube outcomes can produce only local strategy invalidation, exact-host re-sweep, and recheck; only a verified owned Geph backend can produce a restart action; external state produces a warning only. | Active |
| 2026-07-11 | Transparent PF interception is armed only after the local listener and the enabled geo-exit backend have a verified usable port. A runtime geo-exit failure pauses only `com.apple/slipstream`; retries use the native path until a bounded hold expires and backend readiness is re-established. | Active |
| 2026-07-11 | Geph health hysteresis may preserve a previously verified port, but it must never invent an `up` state on cold start. Endpoint failures do not let the tray restart a live Geph process; LaunchAgent `KeepAlive` handles process death, and any future live-process restart must be daemon-coordinated after routing is idle. | Active |
| 2026-07-14 | `Quit Slipstream` keeps its non-disruptive meaning: the daemon and owned Geph LaunchAgent may continue. A separately confirmed tray uninstall disables tray autostart and removes only the verified owned Geph LaunchAgent, private runtime, and Keychain entry before requesting root cleanup. It then removes the root daemon and private PF state, exits the verified tray PID, and removes the exact Slipstream app bundle. A failed ownership check halts cleanup; external Geph, DNS, proxy, PAC, VPN, and PF state are never touched. | Active |
| 2026-07-13 | Root-daemon installation is transactional: enable and bootstrap must be followed by a fresh owned status, exact listener identity, and matching private-PF state. Any failure disables the launchd label, removes the plist and owned runtime, clears only `com.apple/slipstream`, and releases the owned PF token; if token release fails, its record is retained and cleanup fails visibly. An absent or disabled launchd override is durable stop intent: startup and the watchdog never reinstall or re-enable it; only an explicit `Restart Proxy` action may do so. Detached cleanup may signal only a listener PID whose command resolves to the installed Slipstream daemon; an unknown listener is never touched. | Active |
| 2026-07-11 | Diagnostics may distinguish `sidecar_only` when Slipstream's root daemon is absent but its own user Geph LaunchAgent remains loaded. This is a lifecycle fact, not a claim about PF; it contains no PID, paths, hostnames, or secrets. | Active |
| 2026-07-11 | Packaged lifecycle qualification installs the frozen daemon from the built `.app` and preserves an independent PF connection/state through reinstall, restart, and uninstall. Cross-version rollback may begin only from the first release that passes this safety gate; pre-anchor releases are not executable rollback fixtures. | Active |
| 2026-07-11 | World-readable daemon status uses privacy-bounded StatusV2. It exposes aggregate route, backend, environment, and recovery state, never hostname, IP, secret, or raw failure evidence. The tray accepts V1 and V2 during the transition release. | Active |
| 2026-07-11 | Geph exit locations are selectable only from a cached or live city catalog. When neither is available, the tray shows an explicit unavailable state and never invents country-level exits. | Active |
| 2026-07-11 | A geo-exit connection that closes before delivering any payload is a backend failure even if SOCKS CONNECT succeeded. Slipstream pauses only `com.apple/slipstream` before the next client retry; it does not change DNS, proxies, VPNs, or local-bypass routing. | Active |
| 2026-07-12 | An abnormal client-side transport abort or failed downstream write after a local stream pause demotes only the exact generic strategy. A lone orderly EOF remains non-actionable, but two client-first orderly EOFs after at least 15 seconds without downstream progress for the same unknown host within five minutes may take the same exact-host recovery. The next retry may query Xbox DNS directly through its verified DoH endpoint and try the result locally with plain TLS; it also requires two matching clean-EOF signals before being discarded. Discord, YouTube, and other protected classes stay outside this path; it changes no macOS DNS setting and never learns or routes the host through Geph. | Active |
| 2026-07-14 | Every transparent backend relay must settle when either direction completes; direct, local bypass, Smart DNS, and Geph cannot retain a half-open peer task indefinitely. If the client ends first, record the client-first outcome before cancellation; if the server ends first, do not treat cancellation of the client pump as a clean EOF. This lifecycle rule changes no route classification or backend selection. | Active |
| 2026-07-14 | The daemon reserves emergency file descriptors and pauses only `com.apple/slipstream` when a bounded high watermark or `EMFILE`/`ENFILE` is observed. It remains dormant until usage falls below a lower watermark, then may pass the normal backend-readiness gate. Resource pressure must never leave global TCP/443 captured or trigger global PF, DNS, proxy, PAC, or VPN changes. | Active |
| 2026-07-12 | Local-bypass health canaries must use a modern TLS payload handshake with the same scoped fake/desync strategy as a real client. They must not treat rejection of an intentionally minimal synthetic ClientHello as service degradation. Payload timeouts must accommodate measured endpoint latency; this affects health evidence only and never moves Discord or YouTube to Geph. | Active |
| 2026-07-11 | Manual release builds publish only uniquely tagged GitHub prereleases. They are manual-download artifacts and must not replace the stable `releases/latest` updater feed. Stable releases require an exact `v<version>` tag. | Active |
| 2026-07-15 | Every external GitHub Action used by CI and release workflows is registered and pinned to a reviewed immutable commit with a human-readable version label. JavaScript actions use Node 24 releases, JavaScript build jobs use the supported Node 24 LTS instead of EOL Node 20, and the Rust toolchain action is pinned while retaining `stable` as its explicit input default. macOS release jobs install only missing `protoc`, `cmake`, or `pkg-config` formulas with Homebrew auto-update disabled, propagate installation failures, and verify every command afterward; build prerequisites must never be hidden behind `|| true`. | Active |
| 2026-07-15 | Only stable Slipstream app releases may own GitHub's `latest` pointer and updater feed. Preview app releases use unique build-numbered names and never become latest. Vendored Geph artifacts are internal dependency prereleases, never app releases or latest releases, and publishing workflows run with write permissions only from `main`, scheduled runs, or explicit manual dispatch. Repository Actions may create the dependency update PR, but the workflow never approves or merges it; normal `main` protection and required checks still apply. | Active |
| 2026-07-16 | Until Developer ID signing, hardened runtime, notarization, and stapling are implemented, a stable `v*` tag event fails before checkout and cannot publish. Preview publication remains available from `main`. Legacy `v0.1.1` through `v0.1.4` stay immutable and non-prerelease only to preserve the existing `releases/latest` updater endpoint; their titles and notes mark them archival and they are not used for new installs. | Active |
| 2026-07-15 | Python runtime, test, and PyInstaller build graphs are separate, fully pinned lock files generated on Python 3.13 by pinned `pip-tools`. CI, release, and legacy source-install paths install only hash-verified wheels with `--require-hashes` and `--only-binary=:all:`; CI and release jobs select Python 3.13.14. Dependency changes begin in reviewed `.in` files and regenerate all three locks together. | Active |
| 2026-07-15 | macOS application builds target `aarch64-apple-darwin` explicitly. Every published app release carries a deterministic SPDX 2.3 SBOM and a strict artifact manifest bound to the source commit, source commit timestamp, target, byte sizes, and SHA-256 digests. Preview, stable, and `geph-vendor-*` tag namespaces remain distinct; release notes combine a short channel/artifact preface with GitHub-generated merged changes. Existing tags are retained and never repurposed. | Active |
| 2026-07-16 | App releases are dispatched only from `main` or an exact stable tag. After lifecycle and payload verification, GitHub Actions signs SLSA provenance for every release asset and binds the SPDX 2.3 inventory to each app archive or image using the `build-app.yml` OIDC identity. The workflow verifies the signer, source commit, predicate type, and GitHub-hosted runner before publication. | Active |
| 2026-07-16 | Dependency auditing operates on Cargo metadata resolved for the explicit release target rather than every package present in `Cargo.lock`. Pull requests, `main`, scheduled checks, and release builds use a checksum-pinned OSV Scanner. Informational advisories are recorded; every other finding blocks unless an exact package, version, and advisory exception is reviewed and unexpired. Every public audit report is bound to its SBOM, policy, source commit, target, and scanner. | Active |
| 2026-07-16 | A vendored Geph version is identified by its canonical crates.io archive SHA-256, reviewed features and macOS targets, committed `Cargo.lock`, and immutable release revision. Upstream discovery may only open a source-contract PR; binaries are built after merge with `--locked`. Every `geph-vendor-<version>-r<revision>` release publishes a full transitive SPDX inventory and fail-closed audit, plus provenance and SBOM attestations verified again by the app build. Existing tags are never replaced. Audit exceptions are exact, short-lived reachability decisions; incomplete TLS name-constraint enforcement remains explicitly accepted residual risk until upstream updates, not an unreachable-code claim. | Active |
| 2026-07-11 | Preview releases omit remote policy trust keys and signed policy artifacts. Remote policy remains disabled; a stable release must supply and verify the signed policy channel. | Active |
| 2026-07-11 | A preview is offered for installation only after the release workflow runs the privileged PF lifecycle sentinel against that exact built `.app` on a disposable GitHub macOS runner. | Active |
| 2026-07-12 | The tray watchdog does not kickstart a newly installed daemon until it has seen one fresh status snapshot or a bounded startup grace expires. This prevents a healthy daemon from being restarted while it is publishing its first StatusV2 snapshot. | Active |
| 2026-07-12 | A missing durable PF enable token plus a definitively disabled PF state proves an in-memory-only token is stale after owned recovery. Slipstream may clear only that memory and acquire a new private token; persisted, enabled, or indeterminate token states remain fail-closed. | Active |
| 2026-07-12 | Routing changes must pass deterministic data-plane traffic contracts through the real daemon handler. Each contract asserts the intended backend, prohibited backends, and first-plus-streamed payload delivery; it does not mutate PF, DNS, proxy, PAC, VPN, or the live network. | Active |
| 2026-07-12 | A fresh, user-managed Xbox DNS payload proof may be the first geo-exit backend for OpenAI or Anthropic. A runtime Smart DNS miss falls back to owned Geph only when it is verified ready; otherwise only the private PF anchor pauses for a native retry. It never falls into local desync, and Slipstream does not change the system DNS configuration. | Active |
| 2026-07-12 | Raw Telegram MTProto DC traffic is an unmodified safety passthrough and failure signal, not a direct-connect guarantee. When raw MTProto is blocked, the supported user path is the bundled local `tg-ws-proxy`; Telegram never routes through Geph. | Active |
| 2026-07-13 | `billing.openai.com` remains a geo-exit route, but it is not a health canary. A synthetic probe can be refused by an edge or exit without proving that the primary OpenAI flow or Slipstream is unavailable. | Active |
| 2026-07-13 | Tray recovery copy names the affected access class (`local`, `external`, or both) and states that recovery is in progress. It does not surface generic `Needs attention`; raw failure detail remains only in sanitized diagnostics. | Active |
| 2026-07-13 | A live but stale owned Geph may be restarted by the daemon only after the private PF anchor is paused and active Geph sessions reach zero. The daemon must verify the ownership file's user ID against its filesystem owner, the exact `dev.slipstream.geph` LaunchAgent label, and the listener PID/executable/config identity before calling `launchctl kickstart`; unknown listeners and external Geph are never touched. | Active |
| 2026-07-13 | Raw daemon logs and every retained rotation are root-owned regular files with mode `0600`; creation must not follow symlinks. `Open Log` and `Copy Diagnostics` may make a temporary owner-only copy only after an explicit administrator prompt, and diagnostics redact that copy before export. | Active |
| 2026-07-13 | Routing policy and recovery behavior shared across implementations is specified by versioned language-neutral JSON vectors under `contracts/`. Contract v1 is append-only; incompatible behavior changes require a new version so adapters can migrate deliberately. | Active |
| 2026-07-14 | Address racing and route circuit breaking are pure, versioned contracts before they enter any transport adapter. Address attempts use stable family interleaving, a shared deadline, bounded concurrency, and deterministic winner/cancellation rules. Circuit state is scoped to `(service_group, route_class, backend_id)`; opening it suppresses only that candidate and never changes policy, PF, DNS, proxy, PAC, or VPN state. Protected local-bypass groups reject a Geph circuit key. | Active |
| 2026-07-14 | Connection-race orchestration is a pure command/event state machine. It circuit-gates a route before emitting a resolver command, counts an exhausted multi-address race as one logical circuit failure, and ignores late resolver/socket events after a terminal result. A wake delivered exactly at the shared deadline defers timeout by one deterministic turn so a completion timestamped at that deadline can win regardless of queue order; later success is rejected. Scripted adapters execute the shared Python/Rust contract without network I/O; runtime adapters remain a separate change. | Active |
| 2026-07-15 | The Python connection-race I/O adapter owns every resolver task, connector task, timer, and opened stream until one winner is explicitly transferred to its caller. Production `tproxy.py` reaches it only through `connection_probe.py`, after policy and backend selection, and races the existing complete first-payload dialers for local, app-owned Xbox DNS, or freshly proven Smart DNS addresses. It never races route classes or promotes a failure to Geph. Runtime circuit state was deliberately one-shot in this first integration, so one request could not suppress a later request. Cancellation and terminal cleanup close all non-winning streams; PF, launchd, system DNS, proxy, PAC, and VPN behavior are unchanged. | Superseded by the bounded registry below |
| 2026-07-15 | Persistent route-circuit state is applied only to complete backend outcomes, never to an individual IP or desync strategy. One exhausted protected local ladder counts as one `local_engine` failure; freshly proven Smart DNS and verified owned Geph use separate geo-exit keys. An owned-Geph half-open probe recovers as soon as its first downstream payload is delivered, so a healthy long-lived WebSocket cannot retain the sole permit; an early close with zero payload remains failure. Unknown and direct traffic, external Geph, and individual Xbox DNS/local address attempts do not persist suppression. The registry is bounded to 256 non-default entries with a five-minute idle TTL and deterministic LRU; eviction can only retry the same fixed backend, never change route class. Internal registry corruption fails open by clearing suppression, while protected Discord/YouTube keys still reject every Geph edge. | Active |
| 2026-07-13 | Installed lifecycle qualification may suspend and signal only a PID whose command exactly matches the installed daemon. Suspend/resume must exercise the daemon's real wake-gap detection. The diagnostic network-change signal only queues a bounded event; all PF, backend, canary, and status work stays in the normal monitor thread. These stimuli are disposable-CI diagnostics, not user recovery commands, and never mutate DNS, interfaces, proxy, PAC, VPN, or external PF anchors. | Active |
| 2026-07-13 | Packaged tray-crash qualification runs only on disposable CI. It starts the embedded tray as the original non-root user and signals it only when UID and exact executable path match. Fresh HTTPS probes inherit no external proxy environment; the daemon PID, private PF anchor, unrelated PF anchor/state, and persistent connection must survive tray start, crash, restart, and stop. | Active |
| 2026-07-13 | Browser-restart qualification uses the runner's exact Google Chrome executable as the original non-root user with a new owner-only profile for every request. Test-local flags disable Chrome proxy use, background traffic, and QUIC so the request exercises TCP/443 through the qualified PF path; no system DNS, proxy, PAC, VPN, or browser profile is changed. | Active |
| 2026-07-13 | Safari restart qualification uses the system SafariDriver only on disposable CI. The driver runs as the original user on an explicit IPv4 loopback port; the harness uses a direct localhost control connection that cannot inherit proxy settings. Before each request the harness requires no existing Safari process, then verifies the new process by exact UID and executable path. A successful page must report `h2` or `http/1.1`; HTTP/3 cannot prove the qualified TCP/PF path and is rejected without globally blocking QUIC. The isolated WebDriver session is deleted on success or failure; a surviving browser may be signalled only through that verified PID so the next stage proves a fresh process. The harness never enables WebDriver, starts Safari, or changes Safari preferences on a workstation, and it never uses the user's normal browser profile. | Active |
| 2026-07-13 | Account-backed owned-Geph qualification is a protected, main-only manual workflow, never a pull-request secret. It runs as the login user on a disposable macOS runner with the root daemon absent and durably disabled. The harness creates only the exact Slipstream Keychain item and private user runtime, proves a real Steam HTTPS payload through the verified `:9954` listener, crashes the tray, then kills only the revalidated owned Geph PID and requires LaunchAgent `KeepAlive` to replace it. An unrelated `:9909` listener must survive. Cleanup removes only those user-owned artifacts and must leave PF, DNS, proxy, PAC, VPN, and interfaces untouched. | Active |
